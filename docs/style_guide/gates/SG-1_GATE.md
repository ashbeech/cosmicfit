### Phase 1b: Coarse/fine seam

- `ChartAestheticProfile.coarseProfile(fromClusterKey:)` parses `venus_<sign>__moon_<sign>__<element>_dominant` and rejects malformed keys; `coarseProfile(venusSign:moonSign:dominantElement:)` shares the exact same `coarseCore` code path as the fine `derive(from:)`, so register/temperature/finish/metals/formula cannot diverge between coarse and fine (identical by construction, and tested anyway).
- Fine-only refinements: Moon-house orientation adjustment (magnitude 1, can never flip a ±2 pole — exactly how Slate's Moon-in-11th is absorbed), element-margin confidence, stellium-driven overlay policy.

### Phase 1e: Conflict policy

- Decision record: **`docs/style_guide/decisions/profile_conflict_policy.md`** (thresholds, tie-breaks, confidence rules, and the suppress -> neutral -> documented-cache-variant priority order; the coarse lane is never flipped, the formula never altered).

- Mechanical enforcement is the excluded-keyword screen in the overlay generator (priority 1: suppress), with register/orientation text variants as priority 2 (neutral wording). No case required priority 3 (new cache variant).
